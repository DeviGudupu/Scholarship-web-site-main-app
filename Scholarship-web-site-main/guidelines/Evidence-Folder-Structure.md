# EduFund Tracker - Evidence Folder Structure

Create an `evidence` folder in your project root and maintain proofs using this structure.

## Recommended Structure

```text
/evidence
  /01-requirements
    requirements-final.pdf
    scope-in-out.png
    user-stories.png
    acceptance-criteria.png

  /02-prototype
    figma-link.txt
    all-screens-overview.png
    workflow-clickable.mp4
    empty-loading-error-states.png

  /03-flow-navigation
    sitemap-or-flowchart.png
    route-map.png
    role-based-flow.png

  /04-react-implementation
    app-running.png
    routing-proof.png
    form-validation.png
    protected-route-proof.png
    build-success.png

  /05-git-management
    commit-history.png
    branches-prs.png
    contributors-graph.png
    issue-tracking.png

  /06-deployment
    live-link.txt
    homepage-live.png
    route-refresh-proof.png
    mobile-view.png

  /07-linkedin-article
    linkedin-post-link.txt
    article-full-screenshot.png
    section-order-proof.png

  /08-team-coordination
    task-allocation-sheet.png
    meeting-notes.png
    team-chat-updates.png
    member-contribution-summary.png
```

## File Naming Rules
- Use lowercase file names with hyphens.
- Keep names date-friendly when needed: `2026-02-25-build-success.png`.
- Store links in `.txt` files for quick copy during submission.

## Quick Collection Tips
- Capture screenshots immediately after each milestone.
- Keep one proof per rubric point.
- Prefer full-screen captures showing date/time when possible.
- Recheck all public links in incognito mode before final submission.
