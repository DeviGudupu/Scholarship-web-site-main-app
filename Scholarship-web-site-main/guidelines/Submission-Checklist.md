# EduFund Tracker - Submission Checklist (Rubric Based)

Use this checklist before final submission. Target: **Level 6 (10/10)** in all criteria.

## 1) Requirement Finalization (10)
- [ ] Final requirements document completed (scope, objectives, users, modules).
- [ ] In-scope and out-of-scope features clearly listed.
- [ ] User stories / use cases included.
- [ ] Acceptance criteria defined for each major module.
- [ ] Team/mentor validation proof attached (screenshot or message).

## 2) Prototype Design Completeness (10)
- [ ] Figma prototype link is public and accessible.
- [ ] All core screens designed (Landing, Login, Register, Student Dashboard, Search, Scholarship Details, Apply, Tracking, Admin Dashboard).
- [ ] End-to-end workflows are clickable in prototype.
- [ ] Empty, loading, and error states considered.
- [ ] UI consistency maintained (spacing, typography, components).

## 3) Application Flow & Navigation in Prototype (10)
- [ ] Navigation map / user flow documented.
- [ ] Route flow is logical and consistent across all screens.
- [ ] No dead-end pages.
- [ ] Back/forward behavior is clear in flow.
- [ ] Role-based flow separated (student vs admin).

## 4) Front-End Implementation in React (10)
- [ ] React app runs without errors in development.
- [ ] Routing implemented for all key pages.
- [ ] Reusable component structure followed.
- [ ] Forms and validations implemented where required.
- [ ] Protected routes implemented for authenticated/admin pages.
- [ ] App builds successfully (`npm run build`).

## 5) Git Usage & Repository Management (10)
- [ ] Repository is active with meaningful commit messages.
- [ ] Feature-wise commits present (not one bulk commit).
- [ ] Branch/PR workflow used (if team project).
- [ ] Contributions from all team members visible.
- [ ] README and documentation commits included.

## 6) Front-End Deployment (Public Link & Accessibility) (10)
- [ ] App deployed on a public platform (Vercel/Netlify/etc.).
- [ ] Live link works from incognito mode.
- [ ] All main routes are accessible from deployed app.
- [ ] No major runtime/blank page issues.
- [ ] Deployment link added to README and submission document.

## 7) LinkedIn Article (Structure & Completeness) (10)
- [ ] Article is publicly visible.
- [ ] Sections included in order: Problem, Requirements, Prototype, React Build, Git Workflow, Deployment, Learnings, Team Credits.
- [ ] Screenshots/media included for proof.
- [ ] Project GitHub and live links included.
- [ ] Team members mentioned/tagged appropriately.

## 8) Team Coordination (10)
- [ ] Work split documented by module/member.
- [ ] Team progress tracked (meetings/sprint updates/chat summary).
- [ ] Equal participation evidence included.
- [ ] Integration/testing done collaboratively.
- [ ] Final review done by all members.

---

## Final Submission Pack (Minimum)
- [ ] `README.md` updated with setup + live link + repo link.
- [ ] Requirements document (PDF or Markdown).
- [ ] Public Figma prototype link.
- [ ] Public deployed app link.
- [ ] Demo video (3-5 minutes).
- [ ] LinkedIn article URL.
- [ ] Evidence folder with screenshots/proofs.
